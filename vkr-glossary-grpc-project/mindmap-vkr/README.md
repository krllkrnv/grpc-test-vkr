## Описание проекта

Приложение предназначено для создания и управления глоссарием терминов, используемых в выпускной квалификационной работе. Основные возможности:

- CRUD операции с терминами (создание, чтение, обновление, удаление)
- Поиск и фильтрация терминов
- Интерактивная визуализация связей между терминами
- Категоризация терминов
- Связанные термины для построения связей

## Технологический стек

### Frontend

- Vue.js 3

- D3.js
- Vite

### Backend

Backend реализован на gRPC и находится в папке `../glossary-grpc/`:

- gRPC
- Protocol Buffers

- Docker

## Установка и запуск

### Предварительные требования

- Node.js версии 16.0+
- Python версии 3.12+
- Git

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd vkr-glossary-grpc-project
```

### 2. Запуск Backend (gRPC сервисы)

```bash
cd glossary-grpc
docker-compose up -d
```

Backend API будет доступен по адресу: `http://localhost:8000`

Подробная документация по развертыванию backend находится в файле `glossary-grpc/REPORT.md`

### 3. Настройка Frontend

```bash
cd mindmap-vkr
npm install
npm run dev
```

Frontend будет доступен по адресу: `http://localhost:5173`

### 4. Проверка работы

- Frontend: http://localhost:5173
- Backend API: http://localhost:8000
- API документация: http://localhost:8000/docs
- Health check: http://localhost:8000/api/health

Важно: убедитесь, что backend запущен перед запуском frontend.

## Развертывание

### Развертывание Backend

Backend развертывается через Docker. Подробные инструкции находятся в файле `../glossary-grpc/REPORT.md`

### Развертывание Frontend на Vercel

1. Подключение репозитория к Vercel:
   - Зайдите на vercel.com
   - Нажмите "New Project"
   - Подключите GitHub репозиторий
   - Выберите папку `mindmap-vkr` как корневую

2. Настройка сборки:
   - Framework Preset: Vite
   - Build Command: `npm run build`
   - Output Directory: `dist`
   - Install Command: `npm install`

3. Настройка переменных окружения:
   - Убедитесь, что backend развернут и доступен
   - Обновите `API_BASE` в `src/services/api.js` для production окружения

## API документация

### Эндпоинты

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/api/terms` | Получить список терминов с пагинацией |
| GET | `/api/terms/{id}` | Получить конкретный термин |
| POST | `/api/terms` | Создать новый термин |
| PUT | `/api/terms/{id}` | Обновить термин |
| DELETE | `/api/terms/{id}` | Удалить термин |
| GET | `/api/terms/search/{query}` | Поиск терминов |
| GET | `/api/health` | Проверка состояния API |

### Примеры запросов

#### Получение списка терминов

```bash
curl -X GET "http://localhost:8000/api/terms?page=1&per_page=10"
```

#### Создание нового термина

```bash
curl -X POST "http://localhost:8000/api/terms" \
  -H "Content-Type: application/json" \
  -d '{
    "term": "Алгоритм",
    "definition": "Конечная последовательность инструкций для решения задачи",
    "category": "Программирование",
    "related_terms": ["Структура данных", "Сложность"]
  }'
```

## Функциональность

### Основные возможности

1. Управление терминами:
   - Создание новых терминов
   - Редактирование существующих терминов
   - Удаление терминов с подтверждением
   - Просмотр детальной информации

2. Поиск и фильтрация:
   - Поиск по названию термина
   - Фильтрация по категориям
   - Пагинация результатов

3. Визуализация связей:
   - Интерактивная ментальная карта
   - Drag & drop для перемещения узлов
   - Зум и панорамирование
   - Tooltip с информацией о терминах

4. Пользовательский интерфейс:
   - Адаптивный дизайн
   - Интуитивная навигация
   - Подтверждение действий
   - Обработка ошибок

### Технические особенности

- SPA (Single Page Application)
- RESTful API через gRPC прокси
- gRPC Backend на основе gRPC и Protocol Buffers
- CORS поддержка
- Responsive Design
- Error Handling

## Разработка

### Команды для разработки

```bash
# Frontend
npm run dev          # Запуск в режиме разработки
npm run build        # Сборка для продакшена
npm run preview      # Предварительный просмотр сборки

# Backend (gRPC)
cd ../glossary-grpc
docker-compose up -d                  # Запуск через Docker
docker-compose logs -f                # Просмотр логов
```

### Отладка

- Frontend: используйте Vue DevTools в браузере
- Backend: логи выводятся в консоль, API документация доступна по `/docs`
- Network: проверяйте Network tab в DevTools для отладки API запросов
